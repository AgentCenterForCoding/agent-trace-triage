## 新增需求

### 需求:上传Trace数据

用户必须能够通过 Web UI 上传 OTel Trace JSON 文件进行分析。

#### 场景:上传JSON文件

- **当** 用户在 Web UI 选择并上传 .json 文件
- **那么** 系统接收文件并触发定界分析

#### 场景:上传非法文件

- **当** 用户上传非 JSON 文件或格式不合法的 JSON
- **那么** 系统显示错误提示，不进行分析

### 需求:展示Trace瀑布图

系统必须以瀑布图形式可视化展示 Trace 的 Span 结构。

#### 场景:展示正常Trace

- **当** 分析完成
- **那么** 页面展示 Span 瀑布图，显示 Span 名称、层级、耗时、状态

#### 场景:高亮错误Span

- **当** Span 状态为 ERROR
- **那么** 该 Span 在瀑布图中以红色高亮显示

#### 场景:高亮根因Span

- **当** Span 被判定为根因
- **那么** 该 Span 在瀑布图中以特殊标记（如边框加粗）强调

### 需求:展示定界结果

系统必须在 Web UI 明确展示定界结果。

#### 场景:展示归属团队

- **当** 分析完成
- **那么** 页面显示判定的归属团队（agent_team / model_team / mcp_team / skill_team / unknown）

#### 场景:展示置信度

- **当** 分析完成
- **那么** 页面显示置信度（high / medium / low）及颜色标识

#### 场景:展示根因描述

- **当** 分析完成
- **那么** 页面显示 root_cause 字段的人类可读描述

### 需求:展示证据链

系统必须展示从根因 Span 到根 Span 的完整证据链。

#### 场景:列表展示证据链

- **当** 分析完成
- **那么** 页面以列表形式展示 fault_chain 中每个 Span 的名称、层级、状态

#### 场景:点击跳转瀑布图

- **当** 用户点击证据链中某个 Span
- **那么** 瀑布图自动滚动并高亮该 Span

### 需求:加载样本Trace

系统必须提供预置样本 Trace 供用户快速体验。

#### 场景:选择样本Trace

- **当** 用户从下拉列表选择某个样本 Trace
- **那么** 系统自动加载并分析该样本，无需手动上传
