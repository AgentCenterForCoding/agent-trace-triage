## 新增需求

### 需求:加载定界规则

系统必须从 YAML 配置文件加载定界规则。

#### 场景:启动时加载规则
- **当** 系统启动
- **那么** 从 rules.yaml 加载所有定界规则到内存

#### 场景:规则文件格式错误
- **当** rules.yaml 格式不合法
- **那么** 系统启动失败并输出明确错误信息

### 需求:匹配错误Span到规则

系统必须将错误 Span 与规则匹配，找到最佳匹配归属。

#### 场景:匹配Span名称模式
- **当** 错误 Span 名称匹配规则的 span_pattern
- **那么** 该规则成为候选匹配项

#### 场景:无匹配规则
- **当** 错误 Span 不匹配任何规则
- **那么** 归属为 unknown，置信度为 low

### 需求:定位根因Span

系统必须从多个错误 Span 中定位最可能的根因。

#### 场景:单个错误Span
- **当** Trace 中只有一个 status=ERROR 的 Span
- **那么** 该 Span 即为根因

#### 场景:多个错误Span
- **当** 存在多个 status=ERROR 的 Span
- **那么** 选择拓扑深度最大的作为根因

### 需求:生成定界结果

系统必须输出结构化定界结果。

#### 场景:输出定界结果
- **当** 定界完成
- **那么** 输出 owner、confidence、fault_span、fault_chain、root_cause

### 需求:生成证据链

系统必须生成从根因 Span 到根 Span 的完整证据链。

#### 场景:生成证据链
- **当** 定界完成
- **那么** fault_chain 包含从根因沿 parent 链到根的所有 Span
