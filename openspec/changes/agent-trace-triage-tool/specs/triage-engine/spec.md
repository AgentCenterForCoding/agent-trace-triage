## 新增需求

### 需求:加载可配置的定界规则

系统必须从 YAML 配置文件加载定界规则，支持热更新。

#### 场景:加载规则文件

- **当** 系统启动时
- **那么** 从 rules.yaml 加载所有定界规则到内存

#### 场景:规则文件格式错误

- **当** rules.yaml 格式不合法
- **那么** 系统启动失败并输出明确的错误信息

### 需求:匹配错误Span到规则

系统必须将错误 Span 与规则进行匹配，找到最佳匹配的归属规则。

#### 场景:匹配Span名称模式

- **当** 错误 Span 名称匹配规则的 span_pattern
- **那么** 该规则成为候选匹配项

#### 场景:匹配错误类型

- **当** 错误 Span 的 error_type 在规则的 error_type 列表中
- **那么** 该规则的匹配优先级提升

#### 场景:无匹配规则

- **当** 错误 Span 不匹配任何规则
- **那么** 归属判定为 unknown，置信度为 low

### 需求:定位根因Span

系统必须从多个错误 Span 中定位最可能的根因 Span。

#### 场景:单个错误Span

- **当** Trace 中只有一个 status=ERROR 的 Span
- **那么** 该 Span 即为根因 Span

#### 场景:多个错误Span

- **当** Trace 中存在多个 status=ERROR 的 Span
- **那么** 系统选择拓扑深度最大的错误 Span 作为根因

#### 场景:同深度多个错误Span

- **当** 多个错误 Span 拓扑深度相同
- **那么** 系统选择时间最早的错误 Span 作为根因

### 需求:生成定界结果

系统必须输出结构化的定界结果，包含归属、置信度、证据链。

#### 场景:高置信度定界

- **当** 根因 Span 完全匹配某条规则
- **那么** 输出 owner、confidence=high、fault_span、fault_chain

#### 场景:低置信度定界

- **当** 根因 Span 部分匹配或无匹配
- **那么** 输出 owner=unknown、confidence=low、并在 ambiguity 字段说明原因

### 需求:生成证据链

系统必须生成从根因 Span 到根 Span 的完整证据链。

#### 场景:生成正常证据链

- **当** 定界完成
- **那么** fault_chain 包含从根因 Span 沿 parent 链到根 Span 的所有 Span
