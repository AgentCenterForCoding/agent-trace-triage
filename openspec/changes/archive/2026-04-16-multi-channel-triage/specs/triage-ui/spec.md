## 新增需求

### 需求:Trace 上传
用户必须能够通过 UI 上传 Trace 文件进行分析。

#### 场景:拖拽上传
- **当** 用户将 JSON 文件拖拽到上传区域
- **那么** 系统读取文件内容并开始分析

#### 场景:点击上传
- **当** 用户点击上传按钮并选择文件
- **那么** 系统读取文件内容并开始分析

#### 场景:粘贴 JSON
- **当** 用户在文本框中粘贴 JSON 内容
- **那么** 系统解析 JSON 并开始分析

### 需求:实时分析进度
系统通过 SSE 实时展示归因分析进度。

#### 场景:显示进度
- **当** Backend 推送 progress 事件
- **那么** UI 显示当前阶段（解析中 → L1 规则归因 → L2 LLM 归因 → 完成）

#### 场景:分析完成
- **当** Backend 推送 result 事件
- **那么** UI 切换到结果展示视图

#### 场景:分析失败
- **当** Backend 推送 error 事件
- **那么** UI 显示错误信息和重试按钮

### 需求:Span 树可视化
系统必须以树形结构展示 Trace 中的 Span 层级关系。

#### 场景:展示 Span 树
- **当** Trace 解析成功
- **那么** 系统显示可折叠的 Span 树，每个节点显示 name、status、duration

#### 场景:高亮错误 Span
- **当** Span 状态为 ERROR
- **那么** 该 Span 节点以红色高亮显示

#### 场景:高亮根因 Span
- **当** 归因分析完成
- **那么** 被判定为根因的 Span 以特殊样式标记

### 需求:归因结果展示
系统必须清晰展示归因分析结果。

#### 场景:展示责任归属
- **当** 归因分析完成
- **那么** 系统显示 primary_owner、co_responsible、confidence

#### 场景:展示根因说明
- **当** 归因分析完成
- **那么** 系统显示 root_cause 说明和 action_items 列表

#### 场景:展示证据链
- **当** 用户点击"查看证据链"
- **那么** 系统展示从根因 Span 到 Root Span 的完整链路

### 需求:Span 详情查看
用户必须能够查看单个 Span 的详细信息。

#### 场景:点击 Span 节点
- **当** 用户点击 Span 树中的某个节点
- **那么** 右侧面板显示该 Span 的所有属性、事件、时间信息

### 需求:样本 Trace 快速加载
系统必须提供预置的样本 Trace 供用户快速体验。

#### 场景:加载样本
- **当** 用户点击"加载样本"按钮
- **那么** 系统展示可选的样本列表（40 条），用户选择后自动加载并分析

### 需求:设置页面
用户可在 UI 中配置 API Key。

#### 场景:配置 API Key
- **当** 用户在设置页面输入 API Key 并保存
- **那么** 系统调用 Backend 接口持久化 Key

#### 场景:查看当前配置
- **当** 用户打开设置页面
- **那么** 系统显示当前是否已配置 API Key（不显示明文）
