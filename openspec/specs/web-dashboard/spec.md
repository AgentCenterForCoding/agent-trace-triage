# web-dashboard 规范

## 目的
待定 - 由归档变更 agent-trace-triage-tool 创建。归档后请更新目的。
## 需求
### 需求:上传Trace数据

用户必须能通过 Web UI 上传 OTel Trace JSON 文件。

#### 场景:上传JSON文件
- **当** 用户上传 .json 文件
- **那么** 系统接收并触发定界分析

#### 场景:上传非法文件
- **当** 上传非 JSON 或格式不合法
- **那么** 显示错误提示，不进行分析

### 需求:展示Trace瀑布图

系统必须以瀑布图形式展示 Trace 的 Span 结构。

#### 场景:展示瀑布图
- **当** 分析完成
- **那么** 展示 Span 瀑布图（名称、层级、耗时、状态）

#### 场景:高亮错误和根因
- **当** Span 状态为 ERROR 或被判定为根因
- **那么** 以红色高亮或特殊标记强调

### 需求:展示定界结果

系统必须明确展示定界结果。

#### 场景:展示归属和置信度
- **当** 分析完成
- **那么** 显示归属团队、置信度、根因描述

### 需求:展示证据链

系统必须展示完整证据链。

#### 场景:列表展示证据链
- **当** 分析完成
- **那么** 以列表展示 fault_chain 中每个 Span

### 需求:加载样本Trace

系统必须提供预置样本供快速体验。

#### 场景:选择样本
- **当** 用户从下拉列表选择样本 Trace
- **那么** 自动加载并分析，无需手动上传
