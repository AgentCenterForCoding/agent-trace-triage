## 新增需求

### 需求:置信度阈值配置
系统必须支持配置 L2 触发的置信度阈值。

#### 场景:默认阈值
- **当** 未配置阈值
- **那么** 系统必须使用默认阈值 0.8

#### 场景:自定义阈值
- **当** 配置文件中设置 llm_skill.threshold = 0.7
- **那么** 系统必须使用 0.7 作为阈值

#### 场景:阈值范围校验
- **当** 配置的阈值小于 0.0 或大于 1.0
- **那么** 系统必须抛出 ConfigurationError

### 需求:路由决策逻辑
系统必须根据 L1 规则引擎结果决定是否触发 L2 LLM 推理。

#### 场景:低置信度触发 L2
- **当** L1 规则引擎返回 confidence = 0.6
- **且** 阈值配置为 0.8
- **那么** should_invoke_l2() 必须返回 True

#### 场景:高置信度跳过 L2
- **当** L1 规则引擎返回 confidence = 0.9
- **且** 阈值配置为 0.8
- **那么** should_invoke_l2() 必须返回 False

#### 场景:边界值处理
- **当** L1 规则引擎返回 confidence = 0.8（等于阈值）
- **且** 阈值配置为 0.8
- **那么** should_invoke_l2() 必须返回 False（等于阈值时不触发）

### 需求:强制 L2 模式
系统必须支持强制启用 L2 模式（跳过路由判断）。

#### 场景:强制 L2
- **当** 调用方设置 force_l2 = True
- **那么** 系统必须直接触发 L2 推理，不检查 L1 confidence

### 需求:L2 禁用模式
系统必须支持完全禁用 L2（仅使用 L1 规则引擎）。

#### 场景:禁用 L2
- **当** 配置 llm_skill.enabled = False
- **那么** should_invoke_l2() 必须始终返回 False
- **且** 系统不得尝试调用 LLM API
