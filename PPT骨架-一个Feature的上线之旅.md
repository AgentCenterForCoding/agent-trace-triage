# 一个 Feature 的上线之旅 · PPT 骨架

> 两页结构:**主页讲"为什么变" + 附页讲"变成了什么样子"**。
> 主页全景呈现流程/组织/人员三维形变及其驱动力;附页放大流程维度,并把人员能力锚定到流程时间轴上,让三维在同一张图上耦合。

### 主页的设计要点

**1. 顶部"驱动力条"是压力源,不是装饰**
四个能力点(并行 / 可执行 Spec / coding 接管 / 实时评估)用四条向下的箭头打到三栏。读者一眼看到:变化的根源在这里,三维形变是被这股力推出来的。

**2. 每栏中间的"机制"层用统一的"压力 + 出路"模板**
不再是简单的"当前 → 终态"两格对比,而是 **当前 → 机制 → 终态** 三段。机制层强制写成两条因果:
- **压力**:为什么旧形态撑不住了(逼迫)
- **出路**:为什么新形态变得可行(使能)
- 两者用 ∧ 合流到结论

读者一旦在第一栏理解了这个范式,后两栏会自动套上去——一页 PPT 的认知负担反而下降。

**3. 三栏之间的横向"耦合传导"箭头**
表达三维不是各自独立演化,而是相互推动:流程并行化 → 不需要那么多人 → 人员上移 → 才有人能做 Spec 撰写和 Agent 编排。

**4. 人员能力栏的终态描述锚定到流程位置**
不只列能力清单,而是把每种能力锚定到附页时间轴的具体位置(收敛①②/实现期/闭环)——主页和附页之间形成显式的引用链。

### 主页视觉强调建议

- **顶部驱动力条**:高饱和色(深蓝/红),让"力"的感觉强
- **三个机制框**:浅底色 + 手写体注释样式,传达"这是推导过程"
- **三个 ∧ 符号**:放大加粗,是"压力 + 出路"合流的视觉锚点
- **横向耦合箭头**:虚线 + 双向,区别于纵向的实线因果箭头
- **底部内核区**:单独的浅灰底色块,与上方物理上分隔,像"结论盒"

### 附页的三层叠加结构

附页的主视觉是一张**时间-空间图**(横轴时间,纵轴并行轨道),关键是它**把三维同时锚定在同一张图上**:

| 叠加层 | 表达内容 | 对应维度 |
|---|---|---|
| 并行轨道 + 收敛点 ▼ | 流程的拓扑结构 | **流程** |
| 持续校验层(横贯) | 自动化部分 | **流程** |
| **能力使用带**(密度块) | 哪种能力在哪段被消费 | **人员** |
| 收敛点旁的"人:XX"标注 | 人介入只在 4 个点 | **组织** |
| 闭环回流(底部弧线) | 上线非终点 | **流程** |

### 收敛点 × 能力 的精确映射

| 时间节点 | 人在做什么 | 主导能力 |
|---|---|---|
| T+0 ~ T+2 · **收敛 ①** 意图判定 | 模糊语音 + 一句话 → Executable PRD;判断价值假设是否成立 | **Spec 撰写** + **价值判断** |
| T+2 ~ T+4 · **收敛 ②** 架构拍板 | PRD → 契约集;识别跨域耦合;定义 fitness function | **跨域抽象** + **Spec 撰写** |
| T+4 ~ T+24 · 并行实现期 | 编排 20-50 Agent、设计上下文、调试 Prompt、监督收敛 | **Agent 编排**(贯穿) |
| T+24 ~ T+30 · **收敛 ③** 风险门禁 | 通常 Agent 自动放行,**只在风险等级高时人介入** | **风险识别**(冷备) |
| T+30 ~ T+36 · 金丝雀 Go/No-Go | 多维实时数据 → 价值假设是否成立、是否 ramp up | **价值判断 + 风险识别** |
| T+∞ · 闭环回流 | 哪些假设失效、为什么、新规约如何写 | **价值判断** + **Spec 撰写** |

### 能力使用带要传达的三件事

1. **能力是脉冲式的,不是均匀的**:Spec 撰写在前端集中爆发,中段几乎不用,闭环时再次激活——吻合"4 个决策点"的承诺,人介入是稀疏的、点状的。
2. **Agent 编排是中段的"主旋律"**:T+4 到 T+24 这 20 小时虽然主要是 Agent 在干活,但**人通过编排能力在场**——读者不会误以为"中段无人值守"。
3. **跨域抽象有一条低强度贯穿带**:架构拍板时高强度爆发,之后以"低强度贯穿"形态存在——暗示了为什么这种能力是"基础能力"而非"专科能力"。

### 附页视觉强调建议

- **三角形收敛点 ▼**:用红色/醒目色,因为"人介入"是这页唯一需要人关注的事件
- **持续校验层**:浅蓝色横条带,半透明覆盖在并行轨道之上,传达"贯穿"
- **能力使用带**:浅黄/浅橙底色,密度块用饱和度区分(深 = 高强度,浅 = 低强度贯穿)
- **闭环回流箭头**:粗体弧线,从右下绕回左上——这是与当前流程**结构上**最不同的部分
- **20-50 Agent 实现**:故意只画 5 条短轨意思一下,然后用"× N"标注——视觉上传达"很多",不要真的画 20 条把页面塞满

## 两页之间的引用链

| 主页提到 | 附页对应 |
|---|---|
| 流程终态:并行收敛 + 持续校验 + 闭环 | 时间-空间图 + 持续校验层 + 闭环回流弧 |
| 组织终态:1-2 高级 + N Agent | 收敛点 ▼ 旁的"人:XX"标注(4 个点) |
| 人员能力 → 锚定收敛①②/实现期/闭环 | 能力使用带的密度块位置 |
| 内核:意图复杂性 + 协同复杂性 | 收敛点的"对齐"动作 + 闭环的反馈结构 |

讲解时的引导路径:
1. **主页**:为什么变 + 三维形变全景(2-3 分钟)
2. **附页**:翻页放大流程,先讲并行+收敛+持续校验+闭环,再落到能力使用带——把"人员能力"这个抽象词,落到时间轴上的红色密度块(2-3 分钟)
3. 回到**主页底部内核**收束(30 秒)

两页形成"全景—放大—收束"的层次,而不是平铺两页等量内容。
